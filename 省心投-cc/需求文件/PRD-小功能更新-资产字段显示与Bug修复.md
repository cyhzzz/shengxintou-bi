### 3.2 功能详细说明

#### 3.2.1 厂商分析表新增资产字段列

**功能描述**：
在厂商分析报表的平台×代理商聚合数据表格中，新增"资产"字段列，展示开户客户在各厂商、渠道和业务模式下的客户资产总计。

**交互流程**：
```
用户操作 → 系统响应 → 结果反馈
```

**输入参数**：
| 参数名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|--------|------|--------|
| platform | String | 否 | 平台筛选 | "腾讯" |
| agency | String | 否 | 代理商筛选 | "量子" |
| business_model | String | 否 | 业务模式筛选 | "直播" |
| start_date | Date | 否 | 开始日期 | "2025-01-01" |
| end_date | Date | 否 | 结束日期 | "2025-01-31" |

**输出结果**：
| 字段名 | 类型 | 说明 |
|--------|------|------|
| platform | String | 平台 |
| agency | String | 代理商 |
| business_model | String | 业务模式 |
| assets | Decimal | 客户资产总计 |

**业务规则**：
- [x] 规则1：资产统计只统计已开户客户的资产
- [x] 规则2：数据来源为 backend_conversions 表，按 platform, agency, business_model 三个维度聚合
- [ ] 规则3：只统计开户客户（is_opened_account=1），不统计未开户客户
- [ ] 异常处理：如果资产数据为空，显示为0

**数据来源**：`daily_notes_metrics_unified` 表的 `customer_assets` 字段

#### 3.2.2 数据概览表Bug修复

**功能描述**：
修复数据概览报表中，当使用代理商维度筛选器筛选后，客户资产、客户创收、服务存量客户资产等字段显示为0的bug。

**当前问题**：
1. 数据概览表请求客户资产相关指标（`customer_assets`, `customer_contribution`, `existing_customers_assets`）
2. 当前API返回的数据中，这些字段值为0
3. 原因分析：可能是后端聚合逻辑在处理代理商筛选时，对数据进行了错误的过滤

**问题根因**：
需要检查后端 `backend/routes/data.py` 中的 `queryData()` 函数，特别关注：
- 代理商筛选逻辑
- 数据聚合逻辑（`update_daily_metrics_unified.py` 脚本）
- 日志输出中的筛选条件

**修复方案**：
1. 检查 `daily_metrics_unified.py` 聚合脚本中，代理商筛选逻辑是否正确
2. 确认筛选条件传递正确
3. 修正数据库查询条件，确保在指定筛选条件下，数据能正确聚合

**交互流程**：
```
用户操作 → 选择代理商筛选 → 系统响应 → 修复显示问题 → 用户满意
```

**输入参数**：
| 参数名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|--------|------|--------|
| platform | String | 否 | 平台筛选 | "腾讯" |
| agencies | Array | 否 | 代理商筛选（可多选） | ["量子", "众联"] |
| business_models | Array | 否 | 业务模式筛选（可多选） | ["直播", "信息流"] |
| start_date | Date | 否 | 开始日期 | "2025-01-01" |
| end_date | Date | 否 | 结束日期 | "2025-01-31" |

**输出结果**：
| 字段名 | 类型 | 说明 |
|--------|------|------|
| success | Boolean | 请求是否成功 |
| data | Object | 数据对象 |
| customer_assets | Decimal | 客户资产 |
| customer_contribution | Decimal | 客户今年创收 |
| existing_customers_assets | Decimal | 服务存量客户资产 |
| ...其他字段 | ... |

**业务规则**：
- [x] 规则1：代理商筛选不影响日期范围查询
- [x] 规则2：必须正确返回筛选范围内的所有数据
- [x] 规则3：筛选条件为空时，返回所有平台的汇总数据
- [ ] 异常处理：如果查询失败，返回错误信息

### 5.2 接口详细说明

#### 5.2.1 查询代理商分析数据

**请求方式**: `POST`

**请求路径**: `/api/v1/agency-analysis`

**请求参数**：
```json
{
  "filters": {
    "platforms": ["腾讯", "抖音"],
    "agencies": ["量子", "众联"],
    "business_models": ["直播"],
    "start_date": "2025-01-01",
    "end_date": "2025-01-31"
  }
}
```

**响应格式**：
```json
{
  "success": true,
  "data": {
    "summary": {
      "platform": "腾讯",
      "agency": "量子",
      "business_model": "直播",
      "assets": 1256789.00
    },
    "trend": {
      "dates": ["2025-01-01", "2025-01-02"],
      "values": [123456.00, 134567.00]
    },
    "chartData": [
      {
        "platform": "腾讯",
        "agency": "量子",
        "business_model": "直播",
        "date": "2025-01-01",
        "metrics": {
          "cost": 1000.00,
          "impressions": 100000,
          "clicks": 100,
          "leads": 10,
          "assets": 125678.00
        }
      }
    ]
  },
  "message": "查询成功"
}
```

**错误响应**：
```json
{
  "success": false,
  "error": "DATA_QUERY_ERROR",
  "message": "数据查询失败"
}
```

**业务逻辑**：
1. 参数校验：日期范围、平台、代理商、业务模式等参数验证
2. 数据处理：从 `daily_notes_metrics_unified` 表聚合客户资产数据
3. 资产计算逻辑：按 platform, agency, business_model 三维度 SUM(customer_assets)
4. 趋势数据：按日期显示资产变化
5. 返回结果格式化

**错误响应**：
```json
{
  "success": false,
```

1. 参数校验
